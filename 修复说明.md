# Nuitka 打包兼容性问题修复说明

## 🔧 已修复的问题

### 1. ✅ 全局异常处理
**文件**: `main.py`
- 添加了 `global_exception_handler()` 函数
- 当发生未捕获异常时，会显示友好的错误对话框
- 包含详细的错误信息和堆栈跟踪

### 2. ✅ `resource_path()` 支持 Nuitka
**文件**: `main.py`
- 添加了对 Nuitka onefile 模式的检测
- 支持 Nuitka 临时目录查找资源
- 添加了多个位置的回退查找逻辑

### 3. ✅ `subprocess.CREATE_NO_WINDOW` 兼容性
**文件**: `main.py`
- 在使用前检查该属性是否存在
- 避免在某些环境下的 AttributeError

### 4. ✅ `os.uname()` 兼容性
**文件**: `main.py`
- 添加了回退到 `platform.node()` 的逻辑
- 确保在 Windows 上也能正常获取主机名

### 5. ✅ curl_cffi 可选依赖
**文件**: `apple_search_api.py`
- 添加了导入检查
- 如果 curl_cffi 不可用，自动回退到普通 requests
- 提高了跨电脑兼容性

### 6. ✅ 图标文件打包
**文件**: `build.bat`, `honor_logo.png`
- 从 ico 创建了 png 文件
- 更新 build.bat 包含两个图标文件
- 确保 UI 在所有电脑上都能正常显示

## 📦 打包说明

在 Windows 命令提示符或 PowerShell 中运行：

```cmd
cd D:\workspace\apple_code
build.bat
```

### 打包配置特点
- ✅ 使用 Nuitka onefile 模式（单 exe 文件）
- ✅ 包含所有依赖和资源文件
- ✅ 禁用控制台窗口（GUI 模式）
- ✅ 自动包含 curl_cffi、PIL、psutil 等关键模块
- ✅ 完善的错误处理和回退机制

## 🐛 常见问题解决

### 问题：在某些电脑上查询失败
**原因**: curl_cffi 二进制兼容性问题
**解决**: 已添加自动回退到普通 requests 的机制

### 问题：程序启动后立即消失
**原因**: 未捕获的异常导致崩溃
**解决**: 已添加全局异常处理，会显示错误对话框

### 问题：找不到图标文件
**原因**: 资源路径问题
**解决**: 已修复 `resource_path()` 支持 Nuitka

### 问题：硬件ID获取失败
**原因**: wmic 命令在某些环境下不可用
**解决**: 已添加安全的 subprocess 调用和多重回退方案
